FolioCraft Docs

A zero-dependency vanilla JavaScript component library for building interactive, responsive portfolio websites. Every component renders to the DOM directly — no React, no build step required.

Installation

Option 1: Script Tag (CDN / Local)

Include the CSS and JS bundle in your HTML:

<link rel="stylesheet" href="dist/foliocraft.css">
<script src="dist/foliocraft.js"></script>

<script>
  const card = new FolioCraft.ProjectCard({
    title: 'My Project',
    description: 'Built with love',
    tags: ['JS', 'CSS']
  });
  document.getElementById('app').appendChild(card.render());
</script>

Option 2: ES Modules

Import individual components for tree-shaking:

import { ProjectCard } from './src/components/ProjectCard.js';
import { ThemeManager } from './src/theme/ThemeManager.js';
import SkillGroup from './src/components/SkillGroup.js';

Option 3: npm

npm install foliocraft
Note: Always include foliocraft.css — it provides all component styles, theme variables, and responsive breakpoints.

Quick Start

Every FolioCraft component follows the same pattern:

  1. Create an instance with an options object
  2. Render it to get a DOM element
  3. Append the element to your page
// 1. Create
const card = new ProjectCard({
  title: 'My App',
  description: 'A cool project',
  tags: ['React', 'Node'],
  links: { demo: 'https://example.com' }
});

// 2. Render → returns an HTMLElement
const element = card.render();

// 3. Append to the page
document.getElementById('my-container').appendChild(element);

// Optional: trigger the reveal animation
card.reveal();

Theming

FolioCraft ships with light and dark themes managed by ThemeManager. It respects the user's system preference, persists the choice to localStorage, and applies CSS custom properties to :root.

import { ThemeManager } from './src/theme/ThemeManager.js';
import { ThemeToggle } from './src/theme/ThemeToggle.js';

const theme = new ThemeManager();

// Programmatic control
theme.setTheme('dark');
theme.toggle();
theme.isDark();  // true

// Listen for changes
theme.on('themeChange', ({ theme, previousTheme }) => {
  console.log(`Switched from ${previousTheme} to ${theme}`);
});

// Drop-in toggle button
new ThemeToggle(theme, {
  container: document.getElementById('toggle-mount')
});

Custom Themes

theme.registerTheme('ocean', {
  '--fc-primary': '#0ea5e9',
  '--fc-background': '#0c1222',
  '--fc-surface': '#162032',
  '--fc-text': '#e0f2fe',
});
theme.setTheme('ocean');

container.appendChild(group.render());

// Sort highest first
group.sortByLevel('desc');

ContactForm

A configurable, accessible contact form with built-in validation, loading states, success/error messages, and real-time inline error feedback.

Options

OptionTypeDefaultDescription
fieldsobject[]name, email, subject, messageCustom field definitions (see below)
submitLabelstring'Send Message'Submit button text
onSubmitasync functionnullCalled with form data on valid submit
showLabelsbooleantrueShow field labels
inlineErrorsbooleantrueShow per-field error messages
successMessagestring'Thank you!...'Message shown on successful submit
errorMessagestring'Please correct...'Message shown on validation failure

Field Definition

{
  name: 'email',         // field name (required)
  type: 'email',         // 'text', 'email', 'textarea', 'select'
  label: 'Email',        // displayed label
  placeholder: 'you@example.com',
  required: true,       // built-in required validation
  email: true,          // built-in email format check
  minLength: 5,        // minimum character length
  maxLength: 200,       // maximum character length
  pattern: /^\d+$/,     // regex pattern validation
  custom: (val) => {   // custom validator — return true or error string
    return val.includes('@') || 'Must contain @';
  }
}

Events

EventPayloadDescription
submitted{ data }Form submitted successfully
submitError{ error }onSubmit threw an error
validationFailed{ fields }Form validation failed
 
const form = new ContactForm({
  onSubmit: async (data) => {
    await fetch('/api/contact', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data)
    });
  },
  successMessage: 'Thanks! I will get back to you soon.'
});

document.getElementById('contact').appendChild(form.render());

// Listen for submissions
form.on('submitted', ({ data }) => {
  console.log('Sent:', data);
});

// Pre-fill data
form.setData({ name: 'Jane', email: 'jane@test.com' });

// Reset the form
form.reset();

EventEmitter

Every component extends EventEmitter, giving it .on(), .off(), .once(), and .emit(). You can also use it standalone:

import { EventEmitter } from './src/core/EventEmitter.js';

const bus = new EventEmitter();

bus.on('data', (payload) => console.log(payload));
bus.once('init', () => console.log('First time only'));
bus.emit('data', { id: 1 });
bus.off('data', handler);
bus.removeAllListeners();

ScrollAnimator

Wraps IntersectionObserver for scroll-triggered animations. Used internally by SkillBar and Timeline, but also available standalone.

import { ScrollAnimator } from './src/core/ScrollAnimator.js';

const animator = new ScrollAnimator({
  threshold: 0.2,       // 20% visible before triggering
  rootMargin: '0px',
  once: true,            // animate only the first time
  animationClass: 'fc-animate-in'
});

// Observe individual elements
animator.observe(document.querySelector('.my-section'));

// Observe all matching elements
animator.observeAll('.fade-in');

// With callback
animator.observe(el, () => {
  el.classList.add('visible');
});

// Cleanup
animator.destroy();

BreakpointManager

Reactive breakpoint detection using matchMedia. Emits events when the viewport crosses a breakpoint boundary.

import BreakpointManager from './src/responsive/BreakpointManager.js';

const bp = new BreakpointManager();

bp.getCurrent();           // 'lg'
bp.getWidth();             // 1024
bp.is('md');               // false
bp.isAtLeast('md');        // true
bp.isBetween('sm', 'lg'); // true

bp.on('change', ({ current, previous, width }) => {
  console.log(`${previous} → ${current} (${width}px)`);
});

// Default breakpoints: xs:0, sm:576, md:768, lg:992, xl:1200, xxl:1400
// Add custom ones:
bp.addBreakpoint('tablet', 640);

FormValidator

Standalone validation engine used by ContactForm. Can be used independently for any form.

import { FormValidator } from './src/forms/FormValidator.js';

const v = new FormValidator();

// Define rules per field
v.addRules('email', { required: true, email: true });
v.addRules('name',  { required: true, minLength: 2 });
v.addRules('age',   { custom: (v) => v >= 18 || 'Must be 18+' });

// Validate a single field
v.validateField('email', 'bad');
// → { valid: false, errors: ['Please enter a valid email address'] }

// Validate all fields at once
v.validateAll({ email: 'a@b.com', name: 'Jo', age: 25 });
// → { valid: true, fields: { email: { valid: true }, ... } }

// Custom error messages
v.setMessages({ required: 'This cannot be blank' });

Built-in validators: required, email, phone, url, minLength, maxLength, pattern, custom.

createPortfolio()

The killer feature. One function call generates an entire portfolio website from a config object — navbar, hero, projects, skills, timeline, contact form, footer, and theme. Everything wired up with smooth scrolling and animations.

import { createPortfolio } from 'foliocraft';

const portfolio = createPortfolio({
  target: '#app',
  theme: 'developer',

  hero: {
    headline: 'I build things for the web.',
    subheadline: 'Full-stack developer & open source contributor.',
    cta: { text: 'View My Work', href: '#fc-projects' },
    socials: [
      { platform: 'github', url: 'https://github.com/you' },
      { platform: 'linkedin', url: 'https://linkedin.com/in/you' },
    ]
  },

  stats: [
    { value: 50, label: 'Projects', suffix: '+' },
    { value: 5,  label: 'Years' },
  ],

  projects: [
    {
      title: 'My App',
      description: 'Something amazing I built.',
      tags: ['React', 'Node.js'],
      links: { demo: 'https://myapp.com', source: 'https://github.com/...' }
    }
  ],

  skills: [{
    title: 'Tech Stack',
    skills: [
      { name: 'JavaScript', level: 95, color: '#f0db4f' },
      { name: 'React', level: 90, color: '#61dafb' },
    ]
  }],

  experience: [
    {
      title: 'Senior Dev',
      subtitle: 'Acme Corp',
      date: '2023',
      dateEnd: 'Present',
      description: 'Leading the frontend team.',
      tags: ['React', 'TypeScript']
    }
  ],

  contact: {
    onSubmit: async (data) => {
      await fetch('/api/contact', {
        method: 'POST',
        body: JSON.stringify(data)
      });
    }
  }
});

// Change theme later
portfolio.setTheme('bold');

// Access individual components
portfolio.getComponent('timeline');

// Clean up
portfolio.destroy();

CSS Variables

FolioCraft is fully themeable via CSS custom properties. Override any variable in your own stylesheet:

/* Override in your CSS */
:root {
  --fc-primary: #6366f1;
  --fc-primary-dark: #4f46e5;
  --fc-background: #fafafa;
  --fc-card-radius: 16px;
  --fc-card-shadow: 0 2px 8px rgba(0,0,0,0.08);
}

/* Component-specific variables */
:root {
  --fc-skill-bar-fill: #6366f1;       /* default bar color */
  --fc-timeline-dot-size: 20px;      /* larger dots */
  --fc-timeline-line-width: 4px;     /* thicker line */
}

Building

The build script bundles all source files into a UMD bundle + CSS file in dist/:

# Build the bundle
pnpm run build

# Output:
#   dist/foliocraft.js   — UMD bundle (all components)
#   dist/foliocraft.css  — compiled stylesheet

# Run tests
pnpm test

# Dev server with live reload
pnpm run dev
UMD bundle: When loaded via <script> tag, all components are available on the global FolioCraft namespace: FolioCraft.ProjectCard, FolioCraft.Timeline, etc.
ES Modules: For modern projects, import directly from src/ for tree-shaking. The UMD bundle includes everything.
